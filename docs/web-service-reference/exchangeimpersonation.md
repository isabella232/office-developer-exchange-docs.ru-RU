# <a name="exchangeimpersonation"></a>ExchangeImpersonation

Элемент **ExchangeImpersonation** используется в загонах SOAP запроса. Когда этот элемент присутствует, вызываемая пытается представить учетную запись, которая содержится в **элементе ExchangeImpersonation.** 
  
[ExchangeImpersonation](exchangeimpersonation.md)
  
```xml
<ExchangeImpersonation>
   <ConnectingSID/>
</ExchangeImpersonation>
```

 **ExchangeImpersonationType**
## <a name="attributes-and-elements"></a>Атрибуты и элементы

В разделах ниже приводится описание атрибутов, дочерних и родительских элементов.
  
### <a name="attributes"></a>Атрибуты

Нет.
  
### <a name="child-elements"></a>Дочерние элементы

|**Элемент**|**Описание**|
|:-----|:-----|
|[ConnectingSID](connectingsid.md) <br/> |Представляет учетную запись для олицетворения при использовании мыла ExchangeImpersonation.  <br/> |
   
### <a name="parent-elements"></a>Родительские элементы

Нет.
  
## <a name="remarks"></a>Заметки

Учетная запись вызова должна иметь право на **ms-exch-impersonation** на сервере клиентского доступа и **на ms-exch-MayImpersonate** прямо на базе данных почтовых ящиков, содержащем почтовый ящик для выдавания себя за пользователя или объекта Active Directory User/Contact. 
  
Схема, описывающая этот элемент, расположена в виртуальном каталоге EWS на компьютере с MicrosoftExchange Server 2007 и установленной ролью сервера клиентского доступа.
