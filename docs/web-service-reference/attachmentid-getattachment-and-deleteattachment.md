# <a name="attachmentid-getattachment-and-deleteattachment"></a>AttachmentId (GetAttachment и DeleteAttachment)

Элемент **AttachmentId** определяет одно вложение. 
  
```xml
<AttachmentId Id="" />
```

 **RequestAttachmentIdType**
## <a name="attributes-and-elements"></a>Атрибуты и элементы

В разделах ниже приводится описание атрибутов, дочерних и родительских элементов.
  
### <a name="attributes"></a>Атрибуты

|**Атрибут**|**Описание**|
|:-----|:-----|
|**Id** <br/> |Указывает идентификатор вложения.  <br/> |
   
### <a name="child-elements"></a>Дочерние элементы

Нет.
  
### <a name="parent-elements"></a>Родительские элементы

|**Элемент**|**Описание**|
|:-----|:-----|
|[AttachmentIds](attachmentids.md) <br/> | Содержит массив идентификаторов вложений.<br/><br/>  Ниже приводится выражение XPath к этому элементу:<br/><br/>`/DeleteAttachment/AttachmentIds`<br/><br/>`/GetAttachment/AttachmentIds` <br/> |
   
## <a name="remarks"></a>Заметки

Схема, описывающая этот элемент, расположена в виртуальном каталоге EWS на компьютере с MicrosoftExchange Server 2007 и установленной ролью сервера клиентского доступа.
